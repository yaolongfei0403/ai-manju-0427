# UX Design Specification ai漫剧0427

**Author:** Admin
**Date:** 2026/04/30

---

## 需求概述

**功能目标**：设计一个独立的AI模型配置管理页面，支持：
- LLM/T2I/I2V 三组模型的增删改查
- 模型端点配置（API地址）
- API密钥管理
- 从后端动态获取模型列表

**背景**：
- 当前模型配置是硬编码在前端代码中的静态列表
- 模型code只是字符串标识符，实际API调用在FastAPI服务端
- 需要一个独立的管理页面替代硬编码配置

---

## 用户故事与使用场景

### 角色
- **平台管理员**：配置全局AI模型、API端点、密钥
- **项目创建者**：在项目创建时选择已配置的模型

### 核心用户故事
1. 作为管理员，我希望添加新的AI模型，以便平台支持更多模型选项
2. 作为管理员，我希望配置模型的API端点和密钥，以便系统能正确调用模型
3. 作为管理员，我希望启用/禁用模型，以便控制可用模型范围
4. 作为管理员，我希望从后端动态获取模型列表，以便前端无需硬编码

## 功能详细设计

### 1. 模型列表 (左侧面板)

| 功能 | 描述 |
|------|------|
| **模型卡片** | 显示模型图标、名称、提供商、状态标签 |
| **状态指示** | 绿色=启用中，灰色=已禁用 |
| **选中状态** | 左侧高亮边框，点击加载详情 |
| **搜索过滤** | 支持按名称/提供商搜索 |
| **添加模型** | 点击打开右侧详情面板（空表单） |

### 2. 模型详情编辑 (右侧面板)

#### 基本信息
- **名称** (name): 必填，用户显示名称，如 "GPT-4o"
- **Code** (code): 必填，唯一标识，如 "gpt4o"，不可修改
- **提供商** (provider): 必填，如 "OpenAI"
- **描述** (description): 可选，简短描述
- **图标** (icon): 可选，图标选择或emoji

#### 端点配置
- **API端点** (endpoint): API地址，如 "https://api.openai.com/v1/chat/completions"
- **模型名称** (modelName): 在API中传递的模型名，如 "gpt-4o"

#### 密钥配置
- **API密钥** (apiKey): 加密存储，管理员可更新
- **密钥有效期** (keyExpiry): 可选，密钥过期提醒

#### 状态控制
- **启用/禁用**: 切换开关，禁用后项目创建时不可见

### 3. Tab切换 (LLM/T2I/I2V)

| Tab | 模型类型 | 特有字段 |
|-----|---------|---------|
| LLM大模型 | llm | temperature, maxTokens, systemPrompt |
| 文生图模型 | t2i | defaultSteps, defaultCfgScale, supportImg2Img |
| 图生视频模型 | i2v | duration, fps, motionIntensity |

### 4. API接口设计

#### GET /api/v1/admin/models
获取所有模型（支持type过滤）
```json
{
  "models": [
    {
      "id": "uuid",
      "type": "llm",
      "code": "gpt4o",
      "name": "GPT-4o",
      "provider": "OpenAI",
      "endpoint": "https://api.openai.com/v1/chat/completions",
      "modelName": "gpt-4o",
      "status": "active"
    }
  ]
}
```

#### POST /api/v1/admin/models
创建新模型

#### PUT /api/v1/admin/models/:id
更新模型

#### DELETE /api/v1/admin/models/:id
删除模型

#### GET /api/v1/models/types
获取模型类型配置（供下拉选择）

## 交互状态设计

### 模型卡片状态
- **默认**: 深色卡片，白色文字
- **选中**: 左侧蓝色边框，背景提亮
- **禁用**: 整体灰色，状态标签显示"已禁用"
- **悬停**: 背景微亮，显示快捷操作图标

### 表单状态
- **新增模式**: 右侧面板显示空表单
- **编辑模式**: 右侧面板填充现有数据
- **保存中**: 按钮显示loading状态
- **保存成功**: Toast提示，列表刷新
- **保存失败**: Toast显示错误信息

### 边界情况
- **空列表**: 显示"暂无可用模型，点击添加" + 添加按钮
- **加载中**: 骨架屏占位
- **网络错误**: 错误提示 + 重试按钮
- **删除确认**: 二次确认弹窗，提示影响范围

## 与现有系统的集成

### 项目创建页集成
- 项目创建页的模型选择器改为从 `/api/v1/admin/models` 动态获取
- 仅显示 `status: "active"` 的模型
- 按 `type` 分类展示（llm/t2i/i2v）

### FastAPI AI服务集成
- 后端存储模型配置到数据库
- AI调用时，根据 project 配置的 model code 查找对应配置
- 调用对应模型的 endpoint + apiKey
