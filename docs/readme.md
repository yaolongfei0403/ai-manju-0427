## 📋 项目概述

本项目是一个**完整的AI漫剧自动化生成平台**，用户上传小说文本后，系统通过大语言模型自动完成分集拆分、角色/场景/道具资产提取，再经文生图模型生成分镜画面，最终由图生视频模型将静态分镜转为动态漫剧短片。

项目以**HTML原型设计**为界面基准，以**《项目详细设计文档》**为开发规范，最终交付一个功能完整、性能稳定、可直接部署使用的生产级系统。

## 🔄 核心业务链路

### 🚀 AI漫剧工厂：核心业务生产流水线 (SOP)

#### **📦 阶段一：立项与基调设定（准备期）**
* **1. 账户接入：** 用户注册/登录 ➔ 校验权限与配额。
* **2. 引擎点火：** 创建项目 ➔ 确立全局引擎配置（设定受众题材、画幅比例、默认美术风格及底层 LLM/T2I/I2V 模型组合）。

#### **🧠 阶段二：剧本解析与资产沉淀（结构化期）**
* **3. 文本导入：** 上传原著小说文件 ➔ 签署免责声明 ➔ 配置 AI 分集拆解策略。
* **4. 智能分集：** `[AI 节点]` LLM 深度阅读与长文本解析 ➔ 生成分集大纲 ➔ `[人工介入]` 审阅并进行合并/拆分的微调。
* **5. 资产抽离：** `[AI 节点]` 并行提取核心要素（角色/场景/道具）。
* **6. 资产具象化：** `[人工介入]` 润色资产 Prompt ➔ `[AI 节点]` 批量生成资产参考图 (T2I) ➔ 确认并沉淀至“全局资产库”。

#### **🖼️ 阶段三：静态分镜生产（图文期）**
* **7. 分镜剧本：** 进入分集工作台 ➔ `[AI 节点]` 自动生成单镜头分镜描述 ➔ 精准关联资产库中的特定角色与场景。
* **8. 画面生成：** `[人工介入]` 调整分镜 Prompt ➔ `[AI 节点]` 调用 T2I 大模型 + IP-Adapter（确保人物一致性）生成分镜图。
* **9. 抽卡决断：** `[人工介入]` 审阅分镜画面 ➔ 执行“采用”或“重抽” ➔ 确认完毕提交至视频枢纽。

#### **🎬 阶段四：动态合成与后制（成片期）**
* **10. 动态配置：** 进入视频工作台 ➔ 为静态分镜配置运动参数（运镜描述、首尾帧过渡策略）。
* **11. 视频延展：** `[AI 节点]` 调用图生视频大模型 (I2V) 唤醒静态画面，生成高帧率视频片段。
* **12. 终剪导出：** 将片段拖入时间轴 (Timeline) ➔ 非线性拖拽排序 ➔ `[人工介入]` 设定转场特效 (淡入淡出等) ➔ **渲染导出最终漫剧成片**。

### 二、系统架构设计

采用**前后端分离 + 异步任务 + 微服务**的混合架构：

- **前端**：React SPA 部署于 CDN/Nginx，通过 REST API 和 WebSocket 与后端通信。
- **后端 BFF/API 层**：FastAPI 提供业务 API，负责用户认证、项目 CRUD、支付集成等。
- **AI 微服务**：独立部署的 FastAPI 服务，专门处理 LLM 调用、T2I/I2V 生成。通过 Redis 消息队列（BullMQ）接收任务，异步执行并上报进度。
- **任务队列**：Redis + BullMQ 解耦长耗时生成任务，支持重试、优先级、并发控制。
- **实时层**：Socket.IO 服务器广播生成进度、协作锁状态，前端订阅房间事件。
- **存储层**：PostgreSQL 主库 + MinIO 对象存储。

**架构简图**：
```
客户端 (React) ── REST/WebSocket ── API 网关 (Kong/Traefik) ── FastAPI (业务 API)
                                                      │
                                                      ├── Redis (队列/缓存)
                                                      ├── PostgreSQL
                                                      ├── MinIO
                                                      └── AI 微服务 (FastAPI + LiteLLM + GPU)
```

### 四、后端技术方案

**4.1 业务 BFF 与 API**

- **主业务层**：Next.js API Routes + tRPC，处理用户认证、权限校验、项目与分集 CRUD、文件上传预处理及积分计费逻辑。
- **数据持久化**：Prisma ORM + PostgreSQL，类型安全的数据库查询与迁移。
- **AI 微服务**：独立 Python FastAPI 服务，专门处理 GPU 密集计算（LLM 分析、图片/视频生成），与 Next.js 服务解耦。
- **通信方式**：Next.js 通过 REST API (API Key 认证) 向 FastAPI 发起同步请求；长耗时任务通过投递到 Redis 队列异步处理。
- **文件存储**：前端文件通过预签名 URL 直接上传至 MinIO，后端仅记录元数据。
- **支付集成**：Stripe Webhook 处理积分充值回调。
- **实时推送**：WebSocket 通过 Socket.io 广播生成进度与协作锁状态。

**4.2 异步任务系统**

- **任务队列**：BullMQ (Node.js)，基于 Redis，支持优先级排列、重试与进度回溯。
- **任务类型**：`LLM_ANALYSIS`（小说分析）、`ASSET_EXTRACTION`（资产提取）、`IMAGE_GENERATION`（批量生图）、`VIDEO_GENERATION`（图生视频）、`VIDEO_EXPORT`（合成导出）。
- **进度同步**：Worker 在执行期间实时更新 Redis 进度键，Next.js API 层读取后通过 WebSocket 推送到前端，避免轮询。

**4.3 AI 推理微服务**

- 独立 FastAPI 服务，可伸缩部署在 GPU 节点，负责实际模型推理。
- 多模型适配：使用 LiteLLM 统一封装不同 LLM Provider 接口；自建适配器封装图片/视频生成模型 IP-Adapter、ControlNet 等控制功能。
- 任务接口：接收 Next.js 发送的同步 HTTP 请求，长任务支持通过回调或消息队列通知主服务任务完成状态。

### 五、AI 服务层设计

**5.1 模型选型策略**

| 阶段 | 主模型 | 备选模型 | 部署方式 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| 小说分析与分集 | GPT-4o / DeepSeek-V4 | Claude 3.5 Sonnet | API | 长文本处理，成本优先，GPT-4o 逻辑性好 |
| 资产 Prompt 生成 | Claude 3.5 Sonnet | DeepSeek-V3 | API | 长文本结构化输出质量要求高 |
| 文生图（批量） | Seedream 4.5 / SDXL | DALLE 3 / 万相 T2I | 自部署 + API | SDXL 自部署降成本，Seedream 质量高 |
| 图生视频 | Runway Gen-3 / Seedance 2.0 | Pika 1.5 / 万相 I2V | API | 配合 IP-Adapter 保持角色一致性 |

**5.2 AI 编排与适配**

- 使用 **LiteLLM** 或自建适配器统一封装不同模型调用，对外提供标准化的 API 接口，进行模型切换、跟踪和成本控制。
- 图像生成适配器集成 **IP-Adapter**、**ControlNet** 等参考图控制模块，确保多分镜间的角色、场景与风格高度统一。
- 整体生成流程严格遵循“**小说分析 → 智能分集 → 资产抽取 → 批量生图 → 图生视频**”的异步流水线架构，各阶段独立模块化，支持失败断点重试和进度可视化。
